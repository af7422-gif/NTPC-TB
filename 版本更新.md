# 📦 NTPC-TB Git 版本更新操作手冊
按Ctrl + Shift + V 以完整瀏覽
---

## 🔰 一、日常更新流程（最常用）

當你修改了程式碼（新增功能 / 修 bug）：

```bash
git add .
git commit -m "新增登入錯誤提示功能"
git push
```

### 說明

| 指令 | 作用 |
|------|------|
| git add . | 將所有修改加入暫存區 |
| git commit | 建立一個版本紀錄 |
| git push | 上傳到 GitHub |

---

## 🏷 二、建立正式版本（Release）

當你覺得：

> 這個版本可以正式部署

請建立版本標籤（Tag）：

```bash
git tag v1.0.0
git push origin v1.0.0
```

### 建議版本命名規則（Semantic Versioning）

```
v主版號.次版號.修補號
```

| 範例 | 意義 |
|------|------|
| v1.0.0 | 第一個正式可部署版本 |
| v1.1.0 | 新增功能 |
| v1.1.1 | 修正 bug |
| v2.0.0 | 架構重大改版 |

---

## 🌿 三、重大改版（使用分支）

當你要：

- 重構架構
- 改資料庫
- 修改登入機制

不要直接在 main 修改。

### 建立新分支

```bash
git checkout -b refactor-auth-system
```

開發完成後：

```bash
git add .
git commit -m "重構登入架構"
git push origin refactor-auth-system
```

最後合併回 main：

```bash
git checkout main
git merge refactor-auth-system
git push
```

---

## 🔄 四、內網部署時的更新流程

在內網伺服器上更新版本：

```bash
git pull
npm install
npm start
```

或使用 PM2：

```bash
pm2 restart app
```

---

## 🧹 五、修改上一個 Commit（Amend）

當你剛 commit 發現漏改一行：

```bash
git add .
git commit --amend
```

⚠️ 已 push 到遠端的版本不建議 amend。

---

## 🔎 六、查看版本紀錄

```bash
git log --oneline
```

查看版本歷史簡潔列表。

---

## 🧪 七、切換到舊版本測試

```bash
git checkout v1.0.0
```

回到主線：

```bash
git checkout main
```

# 🎯 建議開發規則

### 每次修改都要：

- 寫清楚 commit 訊息
- 不要寫「update」「修改」「fix」

### 好的 commit 範例：

```
新增角色權限 middleware
修正 session 過期問題
重構 routes 結構
```

---

# 📌 建議正式開發流程

```
開發功能
→ git add .
→ git commit
→ git push
→ 功能穩定後
→ git tag vX.X.X
```

確保專案能做到：

```bash
git clone
npm install
npm start
```
