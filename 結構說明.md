# 專案資料夾結構說明文件
按Ctrl + Shift + V 以完整瀏覽
本文件說明本專案中 `src`、`public`、`views`
等主要資料夾的設計目的與責任劃分。

------------------------------------------------------------------------

# 一、src 目錄（後端核心區）

`src` 是整個後端系統的核心區域，負責 API、業務邏輯、資料處理與權限控制。

## 1. routes/

**職責：URL 對應層（路由分發）**

負責將 HTTP 請求導向對應的 Controller。 - 不處理業務邏輯 -
不直接操作資料庫 - 只負責「路由對應」

範例： - auth.routes.js - case.routes.js - review.routes.js

------------------------------------------------------------------------

## 2. controllers/

**職責：請求處理層（Request / Response 控制）**

負責： - 接收 request - 呼叫 service - 回傳 response

不應包含複雜業務邏輯。

------------------------------------------------------------------------

## 3. services/

**職責：業務邏輯層（核心決策區）**

這裡是系統的「流程判斷中樞」。

負責： - 流程狀態轉換（pending → approved） - 權限檢查 - 驗證邏輯 - 呼叫
models 進行資料存取

這層是未來功能擴充的重點。

------------------------------------------------------------------------

## 4. models/

**職責：資料存取層（Database Access Layer）**

負責： - CRUD 操作 - SQL / 查詢語句 - 純資料處理

不得包含流程邏輯。

範例： - user.model.js - case.model.js - log.model.js -
question.model.js - answer.model.js

------------------------------------------------------------------------

## 5. database/

**職責：資料庫連線與初始化**

負責： - 建立 DB 連線 - 連線設定 - 連線池管理

應避免寫業務邏輯。

------------------------------------------------------------------------

## 6. middleware/

**職責：中介層控制**

負責： - 身份驗證（JWT / Token） - 角色權限檢查 - 全域錯誤處理

------------------------------------------------------------------------

## 7. utils/

**職責：共用工具函式**

負責： - 回應格式統一 - 共用 helper 函式 - 小型工具邏輯

------------------------------------------------------------------------

# 二、public 目錄（前端靜態資源）

`public` 為前端瀏覽器可直接讀取的靜態資源區。

## 1. js/

每個頁面一個主 JS 檔。

範例： - login.js - caseList.js - applyCase.js - reviewForm.js -
chart.js

建議設計： - api.js（封裝 fetch） - auth.js（處理 token） -
utils.js（前端工具）

------------------------------------------------------------------------

## 2. css/

放置樣式檔案。

------------------------------------------------------------------------

## 3. images/

圖片資源。

------------------------------------------------------------------------

# 三、views 目錄（前端頁面模板）

負責 HTML 頁面顯示。

範例： - login.html - caseList.html - applyCase.html - reviewForm.html -
dashboard.html

設計原則： - HTML 只負責畫面 - 所有邏輯放在 public/js - 不直接呼叫資料庫

------------------------------------------------------------------------

# 四、設計核心原則

1.  單一責任原則（Single Responsibility Principle）
2.  分層設計（Layered Architecture）
3.  業務邏輯與資料存取分離
4.  前後端責任明確劃分
